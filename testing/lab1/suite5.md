# System & Multi-monitor

| ID | Description (описание) | Pre-Conditions (входные условия) | Actions\Steps (шаги) | Expected Results (ожидаемый результат) |
| :--- | :--- | :--- | :--- | :--- |
| **5.1** | Подключение второго физического монитора в режиме Hotplug | Успешно выполнен кейс 1.1, система запущена на ноутбуке/ПК с одним активным монитором (eDP-1). | Подключите второй внешний монитор (HDMI-A-1) через физический кабель. | Внешний монитор мгновенно обнаруживается системой, на нем отрисовывается новый пустой рабочий стол с обоями по умолчанию. |
| **5.2** | Независимое переключение рабочих пространств на двух мониторах | Успешно выполнен кейс 1.1, подключено два монитора (eDP-1 и HDMI-A-1), фокус ввода находится на мониторе HDMI-A-1. | Нажмите комбинацию `Super+Down` для смены рабочего пространства на мониторе HDMI-A-1. | Рабочее пространство переключается только на мониторе HDMI-A-1, при этом изображение на основном мониторе eDP-1 остается полностью неизменным. |
| **5.3** | Перенос окна на другой монитор | Успешно выполнен кейс 1.1, открыто два монитора, на мониторе №1 сфокусировано окно А. | Нажмите клавиатурное сочетание `Super+Shift+Control+Right` (или перетащите окно мышью на соседний экран). | Окно А исчезает с монитора №1 и появляется на мониторе №2 в виде новой активной колонки. |
| **5.4** | Горячая перезагрузка конфигурации при сохранении изменений | Успешно выполнен кейс 1.1, в `config.kdl` открыт текстовый редактор. | 1. Измените значение `gaps 16` на `gaps 40`. 2. Сохраните файл `config.kdl`. | `niri` автоматически отслеживает изменение файла и мгновенно увеличивает отступы между окнами до 40 пикселей без перезапуска сессии. |
| **5.5** | Обработка синтаксической ошибки в KDL с отображением визуального баннера | Успешно выполнен кейс 1.1. | 1. Внесите синтаксическую ошибку в `config.kdl` (например, удалите закрывающую фигурую скобку `}`). 2. Сохраните файл. | В верхней части экрана появляется контрастный визуальный предупреждающий баннер красного цвета с описанием ошибки и номером строки. |
| **5.6** | Запуск и рендеринг классических X11 приложений через Xwayland | Успешно выполнен кейс 1.1, служба `xwayland-satellite` запущен в сессии `niri`. | Запустите X11-зависимое приложение из терминала. | Окно X11-приложения успешно открывается и рендерится как стандартная колонка в ленте окон `niri`, поддерживая ввод мыши и клавиатуры. |
| **5.7** | Использование встроенной интерактивной утилиты скриншотов | Успешно выполнен кейс 1.1, в настроен бинд для вызова встроенной утелиты: `niri msg action screenshot`. | 1. Нажмите клавишу `PrintScreen` (или настроенное сочетание). 2. Выберите интерактивную область экрана. | Изображение выбранной области успешно захватывается, сохраняется в папку снимков экрана по умолчанию. |
| **5.8** | Безопасное завершение сессии niri | Успешно выполнен кейс 1.1, открыто несколько приложений. | Нажмите комбинацию клавиш выхода `Super+Shift+E` (или выполните `niri msg action quit`). | `niri` посылает сигнал корректного закрытия всем процессам приложений, сохраняет текущие конфигурации и безопасно завершает сессию Wayland. |
| **5.9** | Безопасное отключение монитора | Успешно выполнен кейс 1.1, подключен дополнительный монитор HDMI-A-1, на мониторе HDMI-A-1 открыты приложения. | Отключите монитор. | `niri` посылает сигнал переноса рабочего пространства на другой монитор, ниже текущего. |
